# 《AI Agent & Skills 驱动的量化交易工作流》

## 课程介绍文案

## 一、课程整体介绍

**课程名称：**
《AI Agent & Skills 驱动的量化交易工作流》

**主讲老师：** 白乌鸦

---

**课程定位：**

这是一门帮助量化初学者理解“AI agent 如何协助完成量化研究闭环”的入门课程。
课程基于 QuantSkills 与 PandaData 项目，从 AI skill、`AGENTS.md`、渐进式加载等 AI agent 协作概念讲起，再进入 PandaData 数据接入、本地数据管理、因子计算、因子诊断、策略回测、机器学习模型，到研究报告输出。

**适合学员：**

- 刚开始接触量化研究，希望先建立完整学习路径的初学者。
- 已经会一点 Python 和 Pandas，但不知道如何把行情数据、因子、回测和报告串起来的学习者。
- 希望理解 AI coding agent 如何在一个有工程边界的项目中稳定完成量化研究任务的使用者。

**课程形式：**

- 共 9 讲，每讲约 30 分钟左右。
- 建议节奏：10分钟理论讲解，20 分钟项目代码讲解和实验。

---

## 二、课程学习目标

学完本课程后，学员将能够：

- 理解 AI Agent 的原理和应用，包括 Prompt 和 Sklls 的设计原则，渐进式加载的思想和 Context Window 的管理思路
- 通过 AI 驱动整套量化交易工作流，包括数据下载、管理、因子运算、因子检测、组合管理、规则类和机器学习策略的开发和回测、仿真实盘（可选）。

### 第 1 讲｜让 AI 接管第一步：从 0 拉起你的量化研究项目

**这一讲先解决什么问题？**

很多同学一开始会把 AI 当成“更会写代码的搜索框”。但在真实项目里，AI Agent 更像一个可以读项目、调用工具、修改代码并反复验证的协作者。这一讲先把这个协作者的工作方式讲清楚，再把它放进 QuantSpace 和 PandaData 的场景里，看看一份行情数据是怎样进入量化工作流的。

**本讲主要内容：**

- AI Agent 是怎样工作的：读上下文、理解任务、调用工具、编辑文件、运行验证。
- Prompt、Skills 和 `AGENTS.md` 分别解决什么问题：一个负责表达目标，一个负责沉淀能力，一个负责约束项目协作方式。
- 什么是渐进式加载：为什么不是一次性把所有资料都塞给 AI，而是让它按任务逐步读取相关文档和代码。
- Context Window 为什么重要：上下文不是越多越好，真正有用的是“当前任务需要的上下文”。
- QuantSpace 的基本分工：`skills/` 负责通用能力，`strategies/` 负责策略逻辑，`scripts/` 负责把流程串起来。
- PandaData 接口在课程中的位置：它提供原始行情，后面的数据管理、因子计算和回测都从这里开始。

**课堂实验：**

- 让 AI 先阅读项目说明，再解释它会如何使用 `AGENTS.md` 和 `skills/`。
- 画出一条简单的加载路径：先看项目规则，再看相关 skill，再看脚本，最后运行验证。
- 使用 PandaData 下载或导入一份最小行情数据，理解数据从接口进入项目的第一步。

```bash
uv run python scripts/import_panda_data_demo.py \
  --symbol SHSE.600000 \
  --start-date 20230101 \
  --end-date 20231231
```

**这一讲结束后，学员应该能说清楚：**

为什么复杂量化任务不能只靠一句 Prompt；为什么要把经验写成 Skills；为什么 `AGENTS.md` 能帮助 AI 不乱写代码；以及一份 PandaData 行情数据进入项目以后，大致会流向哪里。

### 第 2 讲｜数据别再乱放：搭建你的本地量化数据仓库

**这一讲要解决什么问题？**

数据下载只是第一步。真正做研究时，更重要的是让数据能被稳定读取、复用和追踪。否则每个脚本都自己处理日期、字段和文件路径，后面的因子、回测和报告都会变得很难维护。

**本讲主要内容：**

- 行情数据的频率与复权。
- 为什么量化项目需要统一的数据目录，而不是把 CSV、Parquet 和中间结果随手放。
- `DataManager` 在项目中的角色：它不是策略逻辑，而是数据和研究产物的管理边界。
- Parquet 适合保存行情数据的原因，以及 DuckDB 在本地查询中的作用。
- QuantSpace 中的基本数据约定：`eob` 时间索引、OHLCV 字段、MultiIndex `(symbol, eob)`。
- 如何让 AI 在写策略之前，先检查本地有没有可用数据。

**课堂实验：**

- 一句话，让AI自动保存多个标的的行情数据。

**这一讲结束后，学员应该能说清楚：**

为什么量化研究不是“下载完数据就结束了”；为什么本地数据管理要有统一格式；以及 Parquet、DuckDB、pool 和 DataManager 分别在工作流里解决什么问题。

### 第 3 讲｜把行情变成信号：做出你的第一批量化因子

**这一讲要解决什么问题？**

行情数据本身不会直接告诉我们该买还是该卖。我们需要先把价格、成交量、波动这些原始信息，转化成可以观察、比较和检验的指标与因子。

**本讲主要内容：**

- 什么是指标，什么是因子：指标描述市场状态，因子尝试解释未来收益差异。
- 常见因子思路：动量、趋势、波动、均值回归、成交量变化。
- `skills.compute` 负责什么：把 OHLCV 数据加工成可复用的计算结果。
- 因子为什么要能批量计算：真实研究通常要同时看多个标的、多个参数和多个时间段。
- 因子绩效分析的基本问题：这个因子是否真的和未来收益有关系，还是只是看起来有道理。
- 理解IC、Rank IC 和分组实验。

**课堂实验：**

- 计算一个趋势类因子和一个波动类因子。
- 对因子做一次简单的 ICIR分析 和 分组实验。
- 让 AI 用自然语言解释分析结果：这个因子看起来有没有研究价值，下一步应该怎么查。

**这一讲结束后，学员应该能说清楚：**

一个想法如何变成因子；一个因子为什么需要被检测；以及为什么不能因为一条曲线好看，就直接认为这个因子有效。

---

### 第 4 讲｜让 AI 帮你挖因子：从想法到可复盘实验

**这一讲要解决什么问题？**

当我们已经知道什么是因子之后，真正的问题会变成：如何持续提出新的因子想法，并且不让这些想法散落在聊天记录、临时脚本和零碎文件里。

**本讲主要内容：**

- 如何和 AI 对话生成因子想法：从市场现象、交易假设、数据字段和已有指标出发。
- 为什么要把因子写成可复用模块，而不是只写在 notebook 或临时脚本里。
- 因子配置如何批量管理：名称、参数、输入数据、计算结果和测试结果要能对应起来。
- `skills.research` 在因子筛选和参数扫描中的作用。
- 如何避免“AI 一次生成很多因子，但没有一个能复盘”的问题。
- 因子库的基本整理方式：保留想法来源、计算逻辑、测试结果和适用边界。

**课堂实验：**

- 给 AI 一个市场假设，让它生成 3 个可测试的因子方向。
- 选择其中一个因子，让 AI 落成可复用的计算逻辑。
- 对多个参数做一次简单扫描，比较结果是否稳定。
- 把因子想法、参数和测试结果整理成一份可复盘记录。

**这一讲结束后，学员应该能说清楚：**

AI 可以帮我们更快地产生因子想法，但不能替我们判断因子是否可靠。真正重要的是把“想法、代码、数据、结果”连在一起，让每一次因子实验都能被复盘。

---

### 第 5 讲｜别只看收益曲线：跑一次真正可信的策略回测

**这一讲要解决什么问题？**

因子只是信号，交易需要的是仓位。一个策略是否值得继续研究，不只取决于信号本身，还取决于如何选标的、如何分配权重、如何处理成本，以及回测是否符合时间顺序。

**本讲主要内容：**

- 从因子到信号，再从信号到权重：date x symbol 权重矩阵是什么。
- 组合管理的基本问题：买哪些、买多少、什么时候换、换手有多高。
- 向量化回测的思路：用矩阵和时间序列一次性计算策略表现。
- `VectorBacktester` 的关键参数：交易价格、信号滞后、手续费、滑点和收益计算方式。
- 多因子组合的基本方法：简单加权、筛选排序、风险约束。
- 为什么回测里最容易犯错的是“未来函数”和“低估交易成本”。

**课堂实验：**

- 用前面计算的因子生成一个简单权重表。
- 跑通一次多因子组合回测。
- 改变手续费和滑点，观察策略结果如何变化。
- 让 AI 解释回测结果中收益、回撤、换手和成本之间的关系。

**这一讲结束后，学员应该能说清楚：**

为什么因子结果不能直接等同于策略收益；为什么仓位、换手、成本和信号滞后会决定一套回测是否可信。

---

### 第 6 讲｜看懂策略怎么运转：拆解两类规则交易案例

**这一讲要解决什么问题？**

很多初学者会把策略理解成“某个指标大于多少就买入”。这一讲会通过两个规则类策略，让学员看到一套策略从数据、规则、权重到回测报告的完整结构。

**本讲主要内容：**

- 规则类策略的特点：逻辑清楚、容易复盘、适合建立第一套研究框架。
- 横截面策略案例：在一组标的中做排序、选择和轮动。
- 时间序列策略案例：围绕单个标的判断趋势、均值回归或风险状态。
- 为什么策略逻辑放在 `strategies/`，而不是全部写进 `scripts/`。
- 两类策略的完整流程：

```text
panel OHLCV -> factors/rules -> weights -> VectorBacktester -> metrics
raw OHLCV bars -> features/rules -> weights -> VectorBacktester
```

**课堂实验：**

- 运行横截面 demo，追踪 panel 数据如何变成权重。
- 运行时间序列 demo，追踪单品种特征如何变成仓位。
- 让 AI 对一个规则类策略做逐段解释：数据来自哪里，信号怎么生成，权重怎么形成，回测怎么看。

**这一讲结束后，学员应该能说清楚：**

规则类策略的价值不在于复杂，而在于清楚。学员应该能判断：什么时候是在做横截面比较，什么时候是在做单品种时序判断，以及为什么清晰的策略结构比临时脚本更重要。

---

### 第 7 讲｜机器学习模型的基本原理：先搞懂特征、label 与训练流程

**这一讲要解决什么问题？**

机器学习在量化里很容易被神化。实际上，它不是“把数据丢进去就能赚钱”，而是把特征、label、训练集、测试集和模型训练组织起来。第 7 讲先不急着看复杂案例，而是把机器学习策略最基础的结构讲清楚。

**本讲主要内容：**

- 机器学习策略的基本结构：特征、标签、训练集、测试集、模型、预测、权重。
- 特征是什么：如何把价格、成交量、波动、趋势等信息整理成模型可以学习的输入。
- Label 为什么关键：模型到底在学习什么，首先取决于我们怎样定义 label。
- `ForwardReturnLabelMaker` 和 `TripleBarrierLabelMaker` 的直觉：一个看未来收益，一个把止盈、止损和持有时间放进标签里。
- 训练集和测试集为什么不能混在一起。
- 过拟合、数据泄露和滚动训练的基本概念。

**课堂实验：**

- 从已有行情数据中生成一组简单特征。
- 对同一段数据生成 forward return label 和 triple-barrier label，对比两种 label 代表的含义。
- 让 AI 解释：如果 label 定义变了，模型学到的东西会怎样变化。

**这一讲结束后，学员应该能说清楚：**

机器学习策略首先是一套数据组织方式。学员应该能说清楚：模型吃进去什么，label 在定义什么，为什么训练集和测试集要分开，以及为什么必须警惕过拟合和数据泄露。

---

### 第 8 讲｜将模型变成策略：跑通机器学习策略案例

**这一讲要解决什么问题？**

理解特征和 label 之后，才适合看完整策略案例。这一讲会把机器学习模型放回真实策略流程里，看它如何从预测结果变成排序、仓位和回测结果。

**本讲主要内容：**

- 横截面机器学习策略：用模型帮助一组标的做排序，选择相对更有优势的标的。
- 时间序列机器学习策略：用模型判断单个标的未来一段时间的状态。
- XGBoost rank 策略的基本流程：因子输入、模型排序、权重生成、回测评估。
- Triple-barrier 策略的基本流程：价格特征、三重屏障 label、模型预测、仓位映射。
- 模型输出为什么不能直接等于交易结论，还需要经过权重、成本和风险检查。
- 如何让 AI 解释机器学习策略，而不是只给出“模型效果不错”这种空泛结论。

**课堂实验：**

- 运行一个 XGBoost rank 策略案例，理解它如何把因子变成排序。
- 运行一个 triple-barrier 策略案例，理解 label 如何影响模型行为。
- 让 AI 解释模型输出如何变成策略权重，再交给回测模块核算收益。
- 对比机器学习策略和前面的规则类策略，观察两者在可解释性、复杂度和风险上的差别。

**这一讲结束后，学员应该能说清楚：**

机器学习策略的价值不在于“用了模型”，而在于模型是否在一个清楚的研究流程里工作。学员应该能说清楚：模型预测如何变成仓位，仓位如何进入回测，以及为什么机器学习策略更需要关注过拟合、数据泄露和解释成本。

---

### 第 9 讲｜让研究自动沉淀：生成一份可复盘的策略报告

**这一讲要解决什么问题？**

一套研究如果不能复盘，就很难判断它是否真的有价值。最后一讲把前面所有环节串起来：从因子筛选、策略比较，到生成图表和报告，让研究结果能被自己和别人看懂。

**本讲主要内容：**

- 研究自动化解决什么问题：批量筛选、参数扫描、策略比较和结果归档。
- `skills.report` 如何把回测结果、图表和文字说明整理成报告。
- 一份策略报告至少应该讲清楚什么：数据、假设、参数、成本、结果、风险和下一步。
- 如何让 AI 帮助生成报告，但不让报告变成空泛总结。

**课堂实验：**

- 运行策略脚本生成报告
- 阅读生成的 Markdown 和 HTML 报告。
- 让 AI 根据回测结果写一段策略复盘，并检查它是否说清楚了数据、假设和风险。

**这一讲结束后，学员应该能说清楚：**

为什么只看收益率是不够的；为什么研究报告必须交代数据、假设、成本和风险；以及如何让 AI 帮自己把研究过程沉淀成可复盘材料。

## 四、课程交付与边界

**本课程会交付给学员的内容包括：**

- 一条由 AI 驱动的量化交易学习路径：从数据下载、数据管理、因子计算、因子检测，到组合回测、策略案例和报告输出。
- 对 AI Agent、Prompt、Skills、`AGENTS.md`、渐进式加载和 Context Window 的基础理解。
- 对 QuantSpace 中 `ingest`、`store`、`compute`、`analyze`、`backtest`、`ml`、`research`、`report` 等能力模块的整体认识。
- 可运行的 PandaData 数据导入示例、规则类策略案例、机器学习策略案例和策略报告生成流程。
- 一个适合初学者完成的结业项目：基于 PandaData 或 sample data 的简单多因子组合研究。

**本课程不会刻意追求的内容包括：**

- 不承诺任何策略收益，也不提供投资建议。
- 不系统讲解所有 PandaData 或 PandaAI 接口，只讲课程工作流中真正需要用到的部分。
- 不把机器学习包装成“自动赚钱”的工具，而是把它放在特征、标签、训练和回测的框架里理解。
- 不深入展开生产级实盘交易系统。仿真实盘只作为可选扩展，帮助学员知道下一阶段会遇到什么。
- 不鼓励把研究逻辑写成一次性脚本。课程会反复强调：能复用的能力放到 skills，策略逻辑放到 strategies，脚本只负责串流程。

**建议结业项目：**

基于 PandaData 或 sample data，完成一个简单的 ETF 或期货池多因子组合研究。

项目交付包括：

- 明确数据来源、symbol 列表和时间范围。
- 使用 `DataManager` 读取或保存数据。
- 至少计算 2 个因子或指标。
- 至少完成 1 次因子检测、参数扫描或策略比较。
- 生成 date x symbol 权重矩阵。
- 使用回测模块运行策略，并显式设置手续费和滑点。
- 输出一份能说明数据、假设、结果和风险的研究报告。

**推荐验收标准：**

- 能说清楚策略假设，而不是只展示一条收益曲线。
- 能说清楚数据从哪里来、如何保存、如何进入因子和回测。
- 能说清楚成本、滑点、回撤和换手对策略结果的影响。
- 能让 AI 帮助生成和整理研究内容，但不是盲目接受 AI 的结论。
